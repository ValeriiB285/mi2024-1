Частина 1: Основи графічного дизайну

Композиція

Do/Don't

Розміщення елементів за сіткою, чіткі зони фокусу.

Робити
  • Використовувати модульну сітку.
  • Вирівнювати текстові блоки.
Не робити
  • Стискати блоки без відступів.
  • Змішувати центрування та ліве вирівнювання.

Баланс

Візуальна рівновага

Пропорційний розподіл візуальної ваги, щоб погляд рухався плавно.

Робити
  • Поєднувати великі елементи з «повітрям».
  • Використовувати послідовні відступи.
Не робити
  • Перенавантажувати одну сторону екрана.
  • Додавати декоративні тіні.

Контраст

Видимість

Чітко показує різницю між важливим і другорядним.

Робити
  • Використовувати 1–2 акцентні кольори.
  • Підсилювати контраст розміром і тоном.
Не робити
  • Додавати багато кольорів без функції.
  • Застосовувати низькоконтрастний текст.

Ієрархія

Пріоритет

Розмір, колір, позиція задають порядок читання.

Робити
  • Виділяти заголовок і ключову метрику.
  • Використовувати підзаголовки.
Не робити
  • Однакова вага всіх текстів.
  • Дублювати акценти без потреби.

Колір

Палітра «Степ»

Природні тони: оливковий, пісочний, теракотовий, світло-зелений.

Робити
  • Перевіряти контраст (WCAG AA).
  • Використовувати акцент обмежено.
Не робити
  • Кислотні відтінки, надмір акцентів.
  • Кольори без функціонального сенсу.

Типографіка

Читабельність

Sans-serif, 16–18 px для тексту, до 36 px для H1.

Робити
  • 2 сімейства максимум.
  • Літерспейсинг і міжряддя 1.4–1.6.
Не робити
  • Капслок для абзаців.
  • Низький контраст підписів.

25–40 хв

Дизайн для візуалізації даних

Типи візуальних елементів

  • Діаграми, графіки, інфографіка, таблиці, карти.
  • Адаптація під дані: точність, доступність, контекст.

Принцип адаптації

Тип даних → візуалізація → коментар до точності, шкал і легенди.

Підсумки

Ключові висновки

  • Принципи дизайну = читабельність + акценти.
  • Тип даних диктує тип графіка.
  • Менше шуму, більше сигналу для прийняття рішень.

Ресурси

  • Storytelling with Data — книга + блог.
  • Data Viz Catalog — довідник типів графіків.
  • Шаблони НАТО/ISR (приклади низького шуму).

Питання для обговорення

  • Що прибрати, щоб підсилити сигнал/шум?
  • Як позначити критичні дані, не втрачаючи доступності?
  • Який графік обрати для часових трендів логістики?
